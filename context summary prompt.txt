[ROLE]
너는 이 레포의 “문서 동기화(Documentation Sync) 엔지니어”다.
목표는 docs/ 폴더의 모든 Markdown 문서가 현재 최신 코드와 1:1로 일치하도록 업데이트하는 것이다.
추측/상상으로 문서를 쓰지 말고, 반드시 코드(파일/함수/라우트/CSS/HTML 문자열)를 근거로 고친다.

[GOAL]
1) docs/ 내부 모든 *.md 파일을 스캔하여, 최신 코드 기준으로 내용(설명/스펙/API/UI/실행방법/검증방법)을 업데이트한다.
2) 문서가 “언제, 무엇을 기준으로” 업데이트됐는지 추적 가능하게 만든다.
3) 특히 최근 변경분(예: 교육 1팀 딜체크 메뉴, nowrap/word-break, 메모 버튼/모달, 포맷 규칙, 정렬 규칙 등)이 문서에 누락되지 않게 한다.

[SCOPE]
- 대상: docs/**/*.md (docs 루트 포함)
- 코드 근거: repo 전체(특히 org_tables_v2.html, dashboard/server/*, tests/*, docs 내 계약/가이드 파일)
- 산출물: 기존 md 파일들을 '덮어쓰기 업데이트' (새 문서가 필요하면 생성 가능하나 남발 금지)

[HARD RULES]
- 코드와 불일치하는 문구/예시는 제거하거나 수정한다.
- “~일 것이다/추정” 같은 문구 금지. 확인된 코드 사실만 작성.
- 모든 문서 상단에 아래 블록을 추가/갱신한다.
  ---
  title: <문서 제목 유지 또는 보정>
  last_synced: 2025-12-24
  sync_source: <해당 문서가 참조하는 주요 파일 경로들 3~8개>
  ---
- API/화면/동작 설명에는 항상 “관련 파일/함수/라우트명”을 명시한다.
- 긴 코드 블록은 최소화하고(문서당 1~2개), 대신 파일 경로와 함수명을 안내한다.

[WORKFLOW]
0) 레포 구조 파악
- docs/ 트리 출력
- 주요 실행 엔트리/서버/프론트 파일 위치 파악

1) 문서 인벤토리 생성(내부 작업용)
- docs/**/*.md 목록을 만들고, 각 문서가 다루는 주제를 한 줄로 요약한다.
- 문서를 아래 카테고리로 분류한다(겹치면 주 카테고리 1개만):
  A. 아키텍처/개요
  B. 로컬 실행/배포/운영
  C. 데이터/스냅샷/파이프라인
  D. API 계약/엔드포인트
  E. 프론트 UI/메뉴/UX 스펙
  F. 테스트/품질
  G. 기타(용어집/런북/FAQ)

2) “문서→코드 근거” 매핑표 작성(내부 작업용)
- 각 문서마다 반드시 sync_source(주요 코드 파일 경로)를 3~8개 지정한다.
- 예: 프론트 UI 문서라면 org_tables_v2.html / 관련 API 라우터 파일 / CSS 블록 / 렌더 함수 등

3) 문서별 업데이트 수행(반드시 이 순서로)
각 문서에 대해:
3-1) 문서에 적힌 주장/스펙/경로/명령어/API/스크린을 코드/현재 파일과 대조한다.
3-2) 불일치 항목을 수정하고, 최신 코드의 실제 동작으로 업데이트한다.
3-3) 변경된 규칙/포맷(예: 날짜 YYMMDD, 컬럼명 변경, 줄바꿈 방지 규칙, 버튼 텍스트, 정렬 기준 등)을 문서에 반영한다.
3-4) 문서 하단에 “Verification(검증)” 섹션을 추가/갱신한다:
     - 사람이 3분 안에 확인 가능한 체크리스트 5~10개
     - 필요한 경우 DevTools 확인 포인트(예: 특정 요소의 computed 값 확인) 포함
3-5) 문서 간 중복이 많으면:
     - 상위 문서(개요/가이드)로 모으고
     - 하위 문서에서는 링크로 참조하도록 정리한다(중복 본문 제거).

4) 특히 다음 ‘최신 변경 포인트’는 관련 문서에 반드시 반영
- “교육 1팀 딜체크” 메뉴:
  - 메뉴 위치(메인 4번째), 두 컨테이너(리텐션/신규), SQL 딜 대상, 리텐션 정의(2025 Won + 금액 파싱 성공 >=0)
  - 정렬: orgWon2025Total DESC → 생성일 ASC
  - 컬럼명/표기: 파트/담당자/가능성/예상/메모, 날짜 YYMMDD, 가능성 리스트는 "값1/값2" 형식
  - 줄바꿈 0 정책: table/td/th/button 모두 nowrap + keep-all + overflow-wrap normal + ellipsis, 테이블 가로스크롤
  - 메모: memoCount==0이면 "메모 없음"(버튼 비노출), 있으면 "메모" 버튼, 모달 날짜 YYMMDD
  - 메모 버튼 높이: td line-height와 동일하게(예: inline-flex, height=line-height, padding 상하 0)
- CSS 적용 실패 원인과 해결 패턴:
  - specificity(예: .card h2 vs 전용 클래스), wrapper(.edu1-dealcheck) 전략
  - “스타일이 안 먹을 때 체크리스트”(DOM 클래스 존재/우선순위/캐시/다른 파일)

5) 문서 품질 규칙
- 제목/섹션은 MECE하게 정리(중복 제거).
- 각 문서 1페이지 내에서 “목적→핵심 개념→사용법→검증→트러블슈팅” 흐름 유지.
- 명령어/경로는 실제 레포 기준으로만. 존재하지 않으면 삭제/대체.

[DELIVERABLES]
- 변경된 모든 docs/*.md를 저장한다.
- 최종적으로 docs/README.md(또는 인덱스 문서)에 “문서 맵(문서 목록 + 1줄 설명 + 링크)”를 최신화한다.
- git diff 기준으로 의미 없는 공백 변경은 최소화한다.

[SELF-CHECK BEFORE FINISH]
- 문서에 나온 파일 경로가 실제 존재하는지 확인
- 문서에 나온 API 경로가 실제 라우터에 존재하는지 확인
- 교육 1팀 딜체크 관련 변경점이 최소 1개 문서(E 카테고리)에 상세히 반영됐는지 확인
- Verification 체크리스트가 문서마다 존재하는지 확인
