# target_attainment_user (v2)
아래 입력을 보고 2026 target 달성 가능성과 “카운터파티 현황 진단(1문단)”을 작성해줘.

입력 해석 규칙:
- won_group_json_compact에는 선택된 상위 조직(카운터파티) 단위의 people/deal/memo 요약 정보가 들어있다.
- 판단은 "현재 실적(actual) + 남은 파이프라인/딜 신호(있다면)"를 기반으로 한다.
- 딜/메모에서 다음 신호를 우선 탐색해 근거로 써라:
  1) 최근성(최근 90~180일)  2) 금액 규모/반복 여부  3) 전환점(포맷 변화, 연간화, 조달/구매 개입, 프로젝트화)
- 불확실하면 “가설” 톤으로 diagnosis를 쓰되, next_actions 첫 번째에 ‘가장 먼저 확인할 질문 1개’를 반드시 포함해라.

diagnosis 작성 템플릿(반드시 3~5문장, 줄바꿈 금지):
1문장: target/actual/gap(또는 달성률)로 현재 위치 규정
2~3문장: 파이프라인/딜/메모에서 읽히는 ‘진척 신호 vs 막힘 신호’ 요약(대표 근거 1~2개 포함)
마지막 문장: 90일 내 만들어야 할 첫 신호(파일럿/예산 구조/운영지표/조달 프로세스 등)와 그 이유

next_actions 작성 규칙:
- 3~5개로, 가능하면 첫 3개는 서로 다른 카테고리 라벨을 붙여라:
  1) [Vehicle] 계약/예산 구조(연간화, 선결제, 범위/좌석 증액 등)
  2) [Value] 제안물/프로그램 설계(파일럿, 운영안, 제안서 아웃라인 등)
  3) [Proof] 증거/락인(리포트, 평가/인증, 레퍼런스, 내부 확산 구조 등)
- 각 액션은 1줄로 간결하게: “누구에게 무엇을 언제까지 확정” 형태 권장

target_2026: {target_2026}
actual_2026: {actual_2026}
