아래 Context는 compact-info-md/v1.1 형식의 마크다운이다. 표와 섹션에서 근거를 찾아 2026 target 달성 관점의 진단과 액션을 제시하라.

입력 해석 규칙:
- Context에는 선택된 상위 조직(카운터파티)의 people/deal/memo 요약이 마크다운 표로 정리되어 있다.
- (truncated due to size limit)가 보이면 잘린 상태이므로, 보이는 정보만 근거로 사용한다.
- 판단은 "현재 실적(actual) + 남은 파이프라인/딜 신호"를 기반으로 한다.
- 딜/메모 신호 우선순위: 1) 최근성(90~180일)  2) 금액 규모/반복  3) 전환점(포맷 변화, 연간화, 조달/구매 개입, 프로젝트화)

diagnosis 작성(줄바꿈 없이 3~5문장):
1문장: target/actual/gap(또는 달성률)로 현재 위치 규정
2~3문장: 표/메모에서 읽히는 ‘진척 신호 vs 막힘 신호’ 요약(대표 근거 1~2개 포함)
마지막 문장: 90일 내 만들어야 할 첫 신호(파일럿/예산 구조/운영지표/조달 프로세스 등)와 이유

next_actions 작성 규칙:
- 3~5개, 가능하면 서로 다른 카테고리 라벨 사용([Vehicle]/[Value]/[Proof] 등)
- 각 액션은 “누구에게 무엇을 언제까지 확정” 형태로 간결히.

출력은 반드시 유효한 JSON 오브젝트 1개만 허용(마크다운/설명/코드펜스 금지).
