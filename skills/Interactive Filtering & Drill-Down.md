# Interactive Filtering & Drill-Down

## 이 스킬을 언제 사용하나요

다음과 같은 경우에 이 스킬을 사용합니다:

- 사용자가 "특정 조건으로 데이터를 좁혀서 보고 싶어"라고 할 때
- 요약된 숫자를 클릭했을 때 상세 내역을 보여달라고 할 때
- 필터 UI를 설계하거나 검색 기능을 구현할 때
- 상세 정보를 모달이나 별도 페이지로 연결해야 할 때

## 주요 기능

### 1. 효율적인 필터링 UX (Smart Filtering)

- **칩(Chip) UI:** 적용된 필터 조건을 상단에 칩 형태로 노출하여 현재 어떤 기준으로 데이터를 보고 있는지 명확히 하고, 원클릭으로 해제할 수 있게 합니다.
- **배치 vs 실시간:** 데이터 양이 많을 경우, 조건을 모두 선택한 후 '적용(Apply)' 버튼을 누르는 방식을 사용하여 불필요한 로딩을 방지합니다.
- **결과 없음 방지:** 필터 선택 시 결과가 0건이 되는 조합은 비활성화하거나, 각 옵션 옆에 예상 결과 건수를 표시합니다.

### 2. 맥락을 유지하는 드릴다운 (Context-Preserving Drill-Down)

- **모달/패널 활용:** 상세 정보를 보기 위해 페이지를 이동하기보다, 모달(Modal)이나 슬라이드 패널(Drawer)을 사용하여 사용자가 현재 대시보드의 맥락을 잃지 않게 합니다.
- **명확한 제목:** 드릴다운된 화면의 헤더에는 클릭한 대상(예: "A기업 상세 내역")을 명확히 표시합니다.
- **필터 상속:** 드릴다운 시, 상위 대시보드에서 적용된 필터(예: 기간, 지역)가 하위 상세 데이터에도 자동으로 적용되도록 합니다.

### 3. 상태 유지 및 복구 (State Persistence)

- **URL 동기화:** 주요 필터링 조건이나 조회 기간을 URL 쿼리 파라미터(예: `?period=Q1&region=seoul`)에 반영하여, 새로고침하거나 링크를 공유해도 동일한 뷰를 볼 수 있게 합니다.
- **뒤로 가기 지원:** 상세 조회 후 닫기 버튼이나 뒤로 가기를 눌렀을 때, 스크롤 위치와 필터 상태가 초기화되지 않고 유지되도록 합니다.

## 사용 예시

**사용자 요청:**
"영업팀별 실적을 보다가 특정 팀을 클릭하면 그 팀의 딜(Deal) 목록이 떴으면 좋겠어. 필터 걸어둔 건 유지하고."

**Codex 적용 결과:**

1.  **인터랙션:** 영업팀 테이블의 행(Row)에 `cursor: pointer` 및 호버 효과 적용. 클릭 시 `TeamDealsModal` 컴포넌트 호출.
2.  **모달 구현:** 모달 헤더에 "1팀 상세 거래 내역" 표시. 배경(Backdrop)은 어둡게 처리하여 집중도 향상. ESC 키로 닫기 지원.
3.  **데이터 처리:** 메인 화면에서 선택된 '2024년 1분기' 필터 조건을 모달의 데이터 요청 API 파라미터로 자동 전달.
4.  **UX 디테일:** 모달을 닫으면 메인 대시보드의 스크롤 위치가 그대로 유지됨.
