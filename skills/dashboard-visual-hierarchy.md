# Dashboard Visual Hierarchy & Layout

## 이 스킬을 언제 사용하나요

다음과 같은 경우에 이 스킬을 사용합니다:

- 사용자가 "대시보드 메인 화면을 잡아줘"라고 요청할 때
- 화면에 너무 많은 정보가 있어 무엇이 중요한지 모르겠다는 피드백이 있을 때
- 다크 모드(Dark Mode) 적용 시 색상 대비나 가독성 문제를 해결해야 할 때
- KPI, 차트, 테이블의 배치를 논리적으로 그룹화해야 할 때

## 주요 기능

### 1. 시각적 우선순위 최적화 (Priority & Grouping)

- **Z-Pattern/F-Pattern 배치:** 가장 중요한 KPI(매출, 이익 등)는 좌상단에 큰 폰트나 카드 형태로 배치하여 즉각적인 인지를 유도합니다.
- **논리적 그룹핑:** 연관된 지표(예: 재무 지표, 운영 지표)는 섹션으로 묶고, 적절한 여백(Whitespace)과 헤더를 사용하여 구분합니다.
- **장식 요소 배제:** 의미 없는 아이콘, 배경 이미지 등 인지 부하(Cognitive Load)를 높이는 요소를 제거하고 데이터 자체를 강조합니다.

### 2. 의미론적 색상 시스템 (Semantic Color System)

- **일관된 상태 색상:** 성공(Green), 경고(Amber/Yellow), 위험/오류(Red) 등 상태를 나타내는 색상은 전체 앱에서 동일하게 사용합니다. 단순 장식용으로 이 색상들을 사용하지 않습니다.
- **카테고리 색상 고정:** 특정 항목(예: '계획'은 파란색, '실적'은 초록색)에 할당된 색상은 차트, 테이블, 범례 전반에서 변하지 않도록 고정합니다.
- **접근성 고려:** 색상만으로 정보를 전달하지 않고, 아이콘이나 텍스트 라벨을 병행하여 색각 이상 사용자도 정보를 파악할 수 있게 합니다.

### 3. 다크 모드 대응 및 명도 대비 (Dark Mode & Contrast)

- **적응형 팔레트:** 다크 모드에서는 완전한 검정(#000000) 대신 짙은 회색(#121212) 배경을 사용하고, 텍스트는 완전한 흰색보다 약간 어두운 오프 화이트(#E0E0E0)를 사용하여 눈의 피로를 줄입니다.
- **채도 조정:** 다크 모드에서는 밝은 배경용 원색보다 채도를 낮추거나 밝기를 조절하여 눈부심(Vibration)을 방지합니다.
- **대비 준수:** 모든 텍스트와 주요 UI 요소는 배경과 최소 4.5:1의 명도 대비를 유지합니다.

## 사용 예시

**사용자 요청:**
"임원용 메인 대시보드인데, 이번 달 매출 현황이 제일 잘 보여야 하고 다크 모드에서도 눈이 안 아프게 해줘."

**Codex 적용 결과:**

1.  **레이아웃:** 최상단 좌측에 '월간 총 매출' 카드를 가장 크게 배치하고, 우측에 전월 대비 증감률을 표시. 하단에는 매출 추이 그래프와 지역별 상세 테이블을 2단 그리드로 배치.
2.  **색상:** '목표 달성' 시 초록색(#4CAF50), '미달' 시 빨간색(#EF5350)을 사용하되, 아이콘(상승/하락 화살표)을 함께 배치.
3.  **다크 모드:** CSS 변수(`--bg-color`, `--card-bg`)를 활용하여 라이트 모드는 흰색 배경, 다크 모드는 #1E1E1E 카드 배경으로 자동 전환되도록 구현. 텍스트 색상은 다크 모드에서 #E0E0E0로 자동 조정.
